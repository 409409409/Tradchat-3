===============================================
TRADCHAT.CSS - COMPLETE STYLE GUIDE & EXPLANATION
===============================================

OVERVIEW
========
This is the main stylesheet for TradChat's welcome page. It's written with extensive comments 
to help beginners understand every CSS rule and creates a warm, traditional aesthetic 
perfect for a conservative social media platform.

THEME & STYLE PHILOSOPHY
=======================
• Color Palette: Warm, earthy tones (off-whites, taupe, charcoal, amber gold)
• Typography: Classic serif headings + clean sans-serif body text
• Vibe: Traditional, trustworthy, cozy, respectful
• Design Principles: Clean layouts, subtle animations, thoughtful spacing

===============================================
SECTION 1: CSS VARIABLES (Design System)
===============================================

The stylesheet uses CSS Custom Properties (variables) for easy theme management:

COLOR VARIABLES:
--color-light: #F5F0E8 (Warm off-white - backgrounds, light cards)
--color-medium: #B8A99A (Muted taupe/tan - borders, subtle fills)  
--color-dark: #2C2416 (Deep charcoal brown - text, nav backgrounds)
--color-accent: #C8860A (Warm amber gold - highlights, buttons, links)
--color-overlay: rgba(44, 36, 22, 0.65) (Semi-transparent dark for modal backdrop)

TYPOGRAPHY VARIABLES:
--font-heading: 'Playfair Display', Georgia, serif (Elegant serif for titles)
--font-body: 'Source Sans 3', Helvetica, sans-serif (Readable sans-serif for content)

ANIMATION VARIABLES:
--transition-speed: 0.5s (Consistent animation timing throughout)

Why this matters: Change any variable here and it updates EVERYWHERE it's used. 
Makes rebranding or theme adjustments incredibly easy.

===============================================
SECTION 2: CSS RESET & BASE STYLES
===============================================

BOX-SIZING RESET:
*, *::before, *::after { box-sizing: border-box; }
→ Prevents padding from making elements wider than expected

MARGIN/PADDING RESET:
body, h1, h2, h3, p, ul, li { margin: 0; padding: 0; }
→ Removes browser defaults for consistent spacing

BASE BODY STYLES:
font-family: var(--font-body) (Clean sans-serif for readability)
background-color: var(--color-light) (Warm off-white background)
color: var(--color-dark) (Dark brown text for contrast)
overflow-x: hidden (Prevents horizontal scrolling)

===============================================
SECTION 3: BUTTON SYSTEM
===============================================

Three distinct button styles with shared base styling:

SHARED BASE STYLES (.btn-sharp, .btn-bubbly, .btn-soft):
• No border (clean look)
• Consistent padding: 10px 22px
• Font: var(--font-body), 14px, weight 600
• Smooth color transitions
• Pointer cursor

BUTTON VARIANTS:
1. .btn-sharp: border-radius: 0px (Hard, modern corners)
2. .btn-bubbly: border-radius: 20px (Pill-shaped, very round)
3. .btn-soft: border-radius: 8px (Gently rounded, "in-between")

This system allows designers to pick the right button personality for each context.

===============================================
SECTION 4: LOGIN MODAL SYSTEM
===============================================

MODAL OVERLAY (.login-modal-overlay):
• Fixed positioning (covers entire viewport)
• Semi-transparent dark backdrop
• Flexbox centering (horizontal + vertical)
• High z-index (1000) to sit above everything
• Smooth fade transitions

MODAL CARD (.login-modal-card):
• White background for contrast
• Rounded corners (12px) for softness
• Generous drop shadow (0 20px 60px rgba(0,0,0,0.3))
• Max width: 380px (comfortable reading width)
• Relative positioning (for close button)
• Slide-up animation on appearance

CLOSE BUTTON (.login-modal-close-x):
• Absolute positioning (top-right corner)
• Transparent background, circular shape
• Hover effect (fills with medium color)
• Smooth transitions

FORM ELEMENTS:
• Labels: Small, bold, dark color
• Inputs: Full width, light gray background, rounded corners
• Focus states: Accent color border highlight
• Submit button: Full width, dark background, accent hover

===============================================
SECTION 7: HERO SECTION
===============================================

HERO CONTAINER (.hero-section):
• Full width, relative positioning
• No extra padding (image fills space)

BACKGROUND IMAGE (.hero-background-image):
• 280px height, full width
• Relative positioning (for text overlay)
• Overflow hidden (contains content)

HERO IMAGE (.hero-placeholder-img):
• object-fit: cover (fills container, maintains aspect ratio)
• object-position: center (centers the image)

WHITE FADE OVERLAY (.hero-background-image::after):
• Pseudo-element with gradient overlay
• Improves text readability over images
• Gradient: 10% white → 50% white → 30% white
• pointer-events: none (allows clicks through)

TEXT OVERLAY (.hero-text-overlay):
• Absolute positioning (bottom-left)
• 32px margins from edges

HEADING STYLES:
• Main heading: 36px, heading font, dark color, white text shadow
• Sub-heading: 15px, weight 600, dark color, white text shadow

===============================================
SECTION 10: RESPONSIVE DESIGN
===============================================

MOBILE BREAKPOINT (@media max-width: 640px):

GRID ADJUSTMENTS:
• About section: Switches from 2-column to 1-column layout
• Prevents cramped content on small screens

TEXT SCALING:
• Hero heading: 36px → 26px (more readable on mobile)
• Maintains hierarchy while improving readability

DIMENSION ADJUSTMENTS:
• Hero height: 280px → 200px (better mobile proportions)
• Text overlay positioning: 32px → 20px margins

MODAL ADJUSTMENTS:
• Adds 16px horizontal margins (prevents edge-to-edge)
• Reduces padding for smaller screens

This responsive approach ensures the design works beautifully on all devices.
